## Shared-file flashpoints

These files are modified by BOTH sessions. Convention: append-only
edits at the bottom so any merge conflict is a one-line resolution.

| File | A.4 change | Merge strategy |
|---|---|---|
| `dashboard/src/modules/billing/types/index.ts` | Append fenced re-export block `export * from "./user";` at bottom | Whichever session merges second keeps both blocks |
| `dashboard/public/locales/en.json` | Append `page.billing.purchase.*` + `page.billing.my_invoices.*` subtrees | Disjoint key subtrees; JSON merge is trivial |

## Naming convention

- A.3.3 demo components live at `modules/billing/components/` root (admin-visible test surface).
- A.4 production components live at `modules/billing/user/components/` (user-facing purchase flow).
- `trc20-payment-display.tsx` (A.3.3, dumb display) ≠ `trc20-payment-screen.tsx` (A.4, full user flow with polling + countdown). Different names, different dirs — no collision.

## Lessons to remember (cross-session)

- **L-012** — translations drift gate is diff-based. A.4 adds net-new keys under `page.billing.purchase.*` and `page.billing.my_invoices.*` in `en.json` + `zh-cn.json` only (other 6 locales untouched to avoid widening baseline drift).
- **L-017** — don't put the literal `t("...")` string in code comments; the drift extractor will grab `...` as a source key.
- Prefer `t("key")` alone in JSX; extract long keys to a `const` if biome would wrap. No 3-arg `t("key", "default")` form.
