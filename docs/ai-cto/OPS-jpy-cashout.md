> 读者:运营方(你),不是开发者。
>
> 目的:把 Aegis Panel 收到的 USDT(来自码商 USDT 结算 + 用户直付
> USDT)安全换成 JPY,进你日本个人银行账户。

---

## TL;DR(如果只看一段)

1. **日本交易所 KYC 后**:bitFlyer 或 GMO Coin 是日本合规交易所,开户要地址 + 身份证 + 银行卡,**日本居民 ~3 天,非居民难**
2. **P2P OTC**:Binance Japan P2P 支持 USDT → JPY 直接对手方,KYC 较轻松但对手方信用靠自己挑
3. **税务**:日本把 crypto 换法币视为**雑所得**,年度总额 > 20 万日元需申报,边际税率 5-45%
4. **频率**:每月一次大批量换,不要每次小额换(手续费 + 税务复杂度)
5. **资金安全**:换出的 JPY 进**你的个人日本账户**(即便你非日本居民也能用外国人专用 SMBC/Shinsei 账户);**绝对不**进国内(中国)账户,追查链条最短

**现实建议**(按你当前"非日本居民 + 无实体"条件):

- **首选 Binance Japan P2P**:相对好开户,对非居民也能(有护照 + 合适签证类型就行),P2P 对手方多
- **次选 bitFlyer**:如果你有日本长期居留证或可以找当地朋友代开
- **不要**用 Bybit/OKX 然后银行转 JPY —— 日本 FSA 2024 禁止非持证交易所对日本居民服务,你银行端会被盯

## Binance Japan P2P 完整流程

### 0. 前置

- 注册 Binance Japan 账户(不是全球 Binance,日本版本独立)
- 完成 KYC L1:护照 + 自拍 + 居住地址证明(非 JP 地址也能,但额度受限)
- 绑定日本银行账户(Shinsei / SMBC / 三菱 UFJ / 樱花 —— 外国人开户推荐 Shinsei Power Flex 或 SMBC Prestia)

### 1. 把 USDT 从你钱包送到 Binance

- 钱包 → Withdraw → TRC20 → Binance 充币地址(TRC20)
- 手续费:~1 USDT(固定)
- 到账:1 分钟内,3 confirmations
- ⚠️ 地址确认三遍,错了钱没了

### 2. P2P 挂单卖 USDT

- Binance Japan → P2P → Sell USDT
- 选价格最好的买方(看 "90 days completed orders" + "% completion rate" ≥ 98%)
- 挂单金额 / 放行价:按市价或略低 0.5% 冲量
- **重点**:看买方的付款方式 —— 只接"日本国内银行转账",别接第三方支付
- 创建订单 → 买方付款 → 你收到入账短信 → Binance 后台确认 → 放币

### 3. JPY 提现到你日本银行

- Binance Japan 本身可能是托管 JPY 的,有些版本要 P2P 买方直接
  汇 JPY 给你银行(跳过 Binance 托管)
- 如果是前者:Binance → Withdraw JPY → 你日本银行账户,1-3 小时
- 手续费:0.1-0.5%

### 4. 频率

- **每月一次,批量 50-200 万 JPY 合适**(太频繁触发反洗钱审查;太
  大单笔可能触发强制申报)
- 不要同一天多次换 —— P2P 对手方不想看到你频繁上架,会降低信誉分

## 税务(**务必读,别省**)

### 基础规则(日本住民)

- Crypto 换法币的差价 = **雑所得**
- 年度 > 20 万日元 → 必须做**確定申告**(每年 2 月 16 日 - 3 月 15
  日)
- 税率:总所得越高越重,边际 15% → 45% 之间(住民税另加 10%)
- **USDT → JPY 的买卖价差** 按你 USDT 的成本基础(操作纪录应完整)

### 基础规则(非日本住民)

- 如果你**年内在日本不超过 183 天**:日本税务上你是"非住民",
  只有"国内源泉所得"要交税
- 但:**JPY 落地到日本个人账户**本身会产生记录,银行每 3-5 年给
  税务署报表,金额大会被约
- 长期路径:如果你持续运营,日本税务很可能认定你"事实上在日本从
  事营业",要求你申告。因此**不要长期把运营收款落到日本个人账户
  而不做申告**

### 建议做法

1. 开始运营的第一个月:Excel 记录每次**出售 USDT 的(日期、金
   额、JPY 价、成本基础)** —— 这是算雑所得的原始数据
2. 年底超过 20 万日元 → 找会计师(税理士,一次性申告 ~3-8 万日
   元)帮你做申告
3. 保留 5 年记录(日本税法要求)
4. **不要偷税**:日本税务署对非住民账户的监控比看起来更细,被追
   缴滞纳金可以翻倍

## 资金安全

### 钱包侧

- **热钱包**(TronLink / imToken / OKX Web3):<10000 USDT 可以
  放,方便收款
- **冷钱包**(Ledger Nano / Trezor):> 10000 USDT 必须冷存
- **定期转移**:热钱包累计超过一周运营金,立即冷存
- **私钥永远不存明文**:助记词手写 + 铁板刻录,分两份存两个物理
  地点

### 账户侧

- Binance Japan 启用 2FA + 邮件白名单 + IP 白名单(如果能)
- 日本银行开网银同样 2FA
- **永不用公司 WiFi 登账户** —— 机场场景下的任何共享网络都要当
  作不可信

### 法律侧

- 你操作的所有网络流量本身在日本是合法的,**你的风险在源头(中
  国发法律诉讼) + 落地(日本税务监控)**
- 保留 5 年完整 PaymentEvent 日志 + 换汇记录 —— 将来若被询问,
  证据在你手里 = 主动
- 不要把"机场业务"写在任何外部文件上 —— LinkedIn / 推特 / 朋友
  圈绝不提

## 应急预案

### 场景 1:Binance Japan 账户被冻结

- 常见诱因:反洗钱触发(流水异常 / KYC 信息变更)
- 动作:立即联系客服,提交"资金来源证明"(你会提前备这堆证明 —
  Aegis Panel 的订单记录 + 码商结算凭证 + 钱包交易记录)
- 备用:提前开好 bitFlyer 或 GMO Coin 作为 B 线

### 场景 2:日本银行账户被冻结(更严重)

- 通常是银行接到反洗钱警示,要你去柜台说明
- 带齐上面说的资金来源证明去
- **运营期间永远保持 2-3 家不同银行的账户**,一家冻结可以切另一
  家,用时间换空间

### 场景 3:税务署询问

- **务必如实回答**,不要撒谎
- 带齐 PaymentEvent 导出 + 换汇记录 + 雑所得申告记录
- 如果你之前没申告,主动补交 + 付滞纳金比被发现时更轻
- 找税理士陪同

## 常见错误(别犯)

1. ❌ 所有 USDT 都留在 Binance 账户里懒得提现 —— 交易所爆雷
   (FTX 式)= 你损失全部
2. ❌ 用朋友的日本账户代收 —— 朋友被冻结的麻烦远大于省的一点麻
   烦;朋友可能被认定"协助洗钱"
3. ❌ 一次性提现 500 万日元 —— 过大单笔触发银行 / 国税強制报告
4. ❌ 用公司 WiFi / 机场 WiFi 登录 Binance / 银行网银
5. ❌ 运营满一年了还没做税務申告
6. ❌ 用 Bybit / OKX 等非日本持证交易所,再把 JPY 汇到日本银行
