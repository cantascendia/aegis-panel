## B 阶段(10-200 用户)— 5 周冲刺

### Week 1:基建落地(¥100 + 5h)

| Day | 动作 | 预算 |
|---|---|---|
| Mon | review/merge audit-log 17 PR(可选,B 阶段不强求) | 0 |
| Tue | 买 VPS:KVM,2 vCPU / 2GB RAM,香港或日本 | ¥30-100/月 |
| Tue | 买域名:你打算长期用的(`xxxx.cc` 类好记) | ¥50/年 |
| Wed | 域名 → CF DNS,开 CF Tunnel:`bash deploy/cloudflare/setup.sh` | 0 |
| Wed | 跑 `bash deploy/install.sh`:出 panel + marznode + PG + Redis | 0 |
| Thu | 配 Reality:`/cto-skills sni-selector` 选 SNI,生成 keypair | 0 |
| Thu | 自己当用户配一次客户端,翻一次墙看 YouTube | 0 |
| Fri | 跑 `bash deploy/agpl-selfcheck.sh` 确认合规 | 0 |

**Week 1 退出标准**:你自己用本 panel 翻墙稳定 24 小时无问题。

### Week 2:接码商(¥1000 保证金 + 4h)

| Day | 动作 |
|---|---|
| Mon | TG 群打听码商:"易支付 T+0 USDT 结算 推荐"。**只列 3 家候选** |
| Mon | 按 [`OPS-epay-vendor-guide.md`](./OPS-epay-vendor-guide.md) §1-§3 硬指标过滤 |
| Tue | 选 1 家,充最低保证金(¥500-1000),申请 merchant_id |
| Wed | 接入 panel:dashboard 创建 channel,按 §A.5 跑 ¥0.01 round-trip |
| Wed | 失败了照 §A.5 排错表来,**不要凑合,不通就换码商** |
| Thu | (可选)接 USDT:[`OPS-trc20-runbook.md`](./OPS-trc20-runbook.md) 跑测试网 round-trip |
| Fri | 创建 3 个真实 plan:¥30 月 / ¥80 季 / ¥240 年 |

**Week 2 退出标准**:¥0.01 自付自收 round-trip 跑通,第二天看到结算到账户。

### Week 3:第一波灰度(0 资金 + 5h/周)

| 第 1 个客户 | 你自己 / 你媳妇 / 你最熟的朋友。**让他真付钱,不要送** |
| 第 2-5 个客户 | 你直接认识的人,告诉他们"内测,有 bug 直接微信我" |
| 第 6-20 个客户 | 上述客户的口碑推荐 |

**关键纪律**:
- ✅ 每个客户加微信,**主动**问"用着怎么样" — 这是你之后做产品的金矿
- ✅ 每天看 panel 健康页 + audit log(如果你 merge 了 v0.3)
- ❌ **不**在 TG 公开群发广告,B 阶段最大风险是被工作量淹死
- ❌ **不**给免费试用,真付费才是真信号

**Week 3 退出标准**:有 5 个**主动续费**的付费客户(说明产品被认可,不是人情)。

### Week 4-5:稳定化 + 流程化

| 动作 | 为什么 |
|---|---|
| 写一份 5 行的"客户使用指南"(订阅链接怎么用) | 重复回答消耗你 50% 时间 |
| 配 Telegram bot 告警:节点宕机 / 流量异常 | B 阶段最怕"客户先发现你才发现" |
| 备份脚本:每天 dump `aegis_billing_*` + `users` 到云盘 | 这是真钱数据 |
| 接 USDT 通道(如果 Week 2 没接) | 海外客户 / 抗风控 |
| 第二台 VPS(异国异网) + Ansible 多节点 | 单 VPS 故障 = 全军覆没 |

**B 阶段退出标准**(开始投入做 C):
- 用户数 ≥ 50 付费 **稳定**(连续 4 周 churn rate < 10%)
- 月营收 ≥ ¥3000 net(扣码商手续费 / VPS / 域名后)
- 客服时间 < 10h/周(超过 = 你被工作量绑架了)
- 至少 2 次客户**主动**问"什么时候支持 X"(说明有产品 PMF 信号)

**任一项不达标 = 留在 B 优化,不进 C**。

## B → C 转换决策(达标后启动)

```
B(50+ 用户 / ¥3000+ MRR / < 10h 客服周)
  ↓
触发条件三选一,启动 C 阶段:
  ① 客户规模 ≥ 100 → 必须 RBAC(你 + 1 客服 = 至少 2 admin 分权)
  ② 月营收 ≥ ¥10k → 投入 5-7 周做 C 的 ROI 算得过
  ③ 客户提"功能 X 没有就走" 数据点 ≥ 3 → 产品压力倒逼
```

**任一触发 → 进入 C**。否则继续榨 B 的红利(B 阶段每多一个月 = 你对产品理解多 30%)。

## C 阶段(200-1000 用户)— 5-7 周工程

### 必做(已有代码,merge + 部署)

1. **audit-log v0.3 merge**(本会话 17 PR,1-2 天 review/merge)
2. **RBAC + 管理员分层**(SPEC #104 已 SEALED,~2 周 S-RB session)
   - 必备:你 + 客服分权,客服不能看码商 merchant_key
3. **真实节点 IP 限制 E2E**(代码已 ship,需真节点跑)

### 应做(还要写代码)

4. **XHTTP / Hysteria2 备用通道**(~2-3 周,差异化 #1 抗封进化)
5. **自动化 Upstream 同步**(~1 周,运维成熟必须)
6. **Reality 健康度仪表盘 v2**(实时监测,Prometheus + Grafana)
7. **客户使用文档站**(MkDocs / Docusaurus,~1 周)

### 可做(C 后段考虑)

8. 多语言客户指南(英 / 简体 / 繁体)
9. 退款 / 故障补偿 SOP 文档化
10. 营销页 + 价格页(从售卖角度重新设计)

**C 阶段退出标准**(进 D 企业):
- 用户数 ≥ 500 付费稳定
- 月营收 ≥ ¥30k net
- 至少 3 次"代理商"询问 reseller 模式(D-016 推翻条件)
- 99.5% SLA 维持 ≥ 3 个月

## 红线(任何阶段都不变)

1. **AGPL-3.0 合规**:`bash deploy/agpl-selfcheck.sh` 必须每月跑一次,自动通过
2. **不接非易支付协议码商**:自研适配 = 维护噩梦
3. **不在群里截图含 merchant_key 的 .env**:已 Fernet 加密,但你别再发原文
4. **每周 backup `aegis_billing_*` 表**:真钱数据,丢一次毁所有信任
5. **panel 不暴露 0.0.0.0**:`UVICORN_HOST=127.0.0.1` + CF Tunnel / Nginx,**永远**
6. **用户敏感数据不进 audit log 明文**:`AUDIT_SECRET_KEY` 在 `.env`(已实现)
7. **D-003 法律边界**:不给中国 / 伊朗 / 俄罗斯 IP 提供运营 panel 入口(数据面节点不在此限)
