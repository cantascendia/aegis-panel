# 运营手册 — 易支付码商选择与接入

> 读者:运营方(你),不是开发者。
>
> 目的:选择一家(最好两家)易支付码商,对接 Aegis Panel 的
> `EPayProvider`,并在运营期间管住资金风险。

---

## TL;DR(如果只看一段)

1. **不要只上一家**,最少两家备用,一家挂立即切
2. **首周只放小量流量**(<500 RMB/天)测试 T+0 结算稳定性
3. **永远选 USDT 结算**(不是 RMB 落卡),直接绕过银行封卡
4. **码商保证金越低越好**,1000 RMB 以内都可接受,>5000 的不考虑
5. **跑路是常态**:备份多家 + 永远不压余额 = 能活下来

## 什么是"码商"(背景)

- 码商 = **支付聚合二清商**。他们自己不是支付宝/微信官方,但他们
  聚合了许多普通人的个人支付宝/微信账号,并把"支付能力"打包成一
  个 API 卖给下游(你)
- 他们通过**池化多个账号 + 频繁轮换**来降低单号被封的概率
- 他们赚的是**手续费差价**:收你 2-5%,但因为不用向支付宝报商户
  备案,他们没有合规成本
- 他们的业务**在中国法律属"二清"**,高危但海量存在;执法只抓
  大鱼,你作为"用户"(运营方)通常不是直接目标,但**余额在码商那
  就随时可能清零**

## 你的选择硬指标(按重要性排序)

### 1. T+0 结算 / 可 USDT 结算(致命重要)

- **T+0** = 今天收到的款今天就能从码商后台提现。T+1 已经是红灯,
  T+3 以上不考虑
- **USDT 结算** = 码商把你应得的钱直接打 USDT TRC20 到你钱包,
  绕过支付宝提现 → 银行卡这条链路。封卡风险 = 0
- 没有 USDT 结算的码商,100% 跳过

### 2. 近 90 天 "跑路记录" 清零

- 到 telegram 群 `@xx代收支付吐槽` 或类似黑名单群搜码商名 +
  "跑路" / "冻结" / "不结算"
- **近 90 天有一起未兑付事件就不考虑**,哪怕人家解释得天花乱坠
- 找 3 个使用中的同行交叉验证(machine owners 群)

### 3. 多通道冗余

- 一家码商内部要有多条"通道"(alipay_1, alipay_2, wepay_1, ...)
- 单条通道挂 = 其他通道顶上,你这边不掉单
- 只有 1-2 个通道的小码商,单账号封 = 他们挂 = 你挂

### 4. 保证金要求低

- 0-1000 RMB:健康,不紧张
- 1000-5000:尚可,但要确认他们的**保证金退还历史**
- \>5000:跳过,要么骗子要么把你钱拴住

### 5. API 文档 = 易支付标准

- 问清楚:"你们是 `apiv1.php?act=order` 这种标准接口吗?" 标准
  = `EPayProvider` 现成适配
- 不是标准易支付 → 需要我们写一个 per-vendor 适配,追加工期 +
  维护面。慎选

### 6. 最低手续费 + 无隐藏扣费

- 正常手续费 2-5% 都算合理
- 小心:"保底最低单笔手续费" / "T+0 加收 xx%" / "提现 USDT 加 1%"
  —— 累加下来 7% 都可能

## 市场现状(**仅观察,不是推荐**)

> 以下名单基于公开信息和同行闲聊,不代表任何 endorsement。所有数据
> 都会过时,自己验证。

| 码商模式 | 描述 | 注意 |
|---|---|---|
| 大型聚合(1万+ 商户) | 通道多、跑路概率低、但手续费贵(4-5%) + 保证金高 | 对 >500 用户的成熟机场合适 |
| 中型聚合(几百到几千商户) | 中等价位 + 中等风险 | MVP 阶段性价比最好 |
| 独立码商("皮包") | 个位数老板 + 几十通道,手续费低到 2% | 跑路风险最高,只能作备用或者早期实验 |

**CTO 建议**: 找**两家中型 + 一家独立小型**,备用互相切换。大型没
必要(你刚起步,量不够养他们的保证金)。

## 首次合作"测试周"标准步骤

1. **签约 + 押金最少档** —— 对方要求多少就给多少,但不要提前充
   运营金。保证金进去就抱着"随时会没"的心理预期
2. **Aegis Panel 管理员后台添加码商**,配置 `merchant_code`、
   `merchant_key`、`gateway_url`,**enabled=false**(先关)
3. **手工测试一笔 0.01 RMB 订单**:打开 enabled,你自己扫一笔
   0.01 RMB → 看到 webhook 来 + 订单变 paid + 用户激活 →
   手动申请提现 0.01 RMB(走 USDT)→ 秒到你钱包
4. 如果步骤 3 有一条慢于 5 分钟,打**红旗**,换家
5. **小量投产** (<500 RMB/天)运行 7 天,每天早上打开后台看未
   结算余额,对账
6. 第 7 天**大额提现测试**:申请 3000 RMB USDT,看小时级到账
7. 7 天稳定无异常 → 扩量到 2000 RMB/天,逐步扩大
8. **永远不要在码商后台攒超过 3 天的流水**,T+0 提现立即转走

## 日常运营安全

### 永不违反的准则

- **永不在码商后台留夜**。每天打烊前把余额提现完
- **永不把保证金提到足以影响你生意的数额**。超过 3000 RMB 要三
  思,超过 10000 RMB 绝对不行
- **多家并行**:2-3 家同时开着,每家承载 30-50% 流量,任何一家
  出事不影响主路
- **日志**:每笔订单 + 每次结算 PaymentEvent 表里都有,异常时
  可以拉数据找码商理论

### 出事典型(照这个清单检查)

1. 通道"维护中" 1-2 小时 —— 正常,等
2. 通道"维护中" > 6 小时 —— 把它标 disabled,切到另一家
3. 你面板收到的 `paid` 比你码商后台看到的多 —— 立即停用该通道,
   查是否码商欠结算
4. 提现申请超过 24h 未到账 —— 开始收集证据,同时通知群里朋友
   (他们可能也遇到)
5. 码商客服不回话 —— **跑路征兆**,立即通知所有其他用户暂停这
   条通道的付款

### 应急演练

每月跑一次**码商失联演习**:
- 把所有 EPay 通道 disabled 一个小时
- 观察 TRC20 通道是否顶住了(应该能接住那些懂 crypto 的用户)
- 计算 TRC20-only 模式下的日收入
- 确认自己"即使所有码商全死掉" 还有最低生存线

## 在 Aegis Panel 中的配置位置

`/admin/billing/channels` 页面:
- 每家码商 = 一个 channel row
- 字段:name(显示)、gateway_url、merchant_code、merchant_key、
  enabled、priority(多家并行时的展示顺序)
- 测试按钮:一键发 ¥0.01 订单验证联通性
- 状态灯:最近 1h webhook 送达率;< 80% 变黄;< 50% 变红

## 退出码商的步骤(若决定终止合作)

1. 后台停用 channel → 新订单不再分配到这家
2. 等 48h,让 in-flight 订单全部结清
3. 从码商后台一次性提清所有余额
4. 申请保证金退还(多数要 7-30 天;有的拒,提前认)
5. 从 Aegis Panel 删除 channel(完整保留 PaymentEvent 历史)

## 附录 — 易支付协议字段速查

供你查 Panel 日志、和码商对账用:

| 字段 | 含义 | 由谁产生 |
|---|---|---|
| `pid` | 你的 merchant_id | 码商分配 |
| `out_trade_no` | Aegis Panel 的 invoice.id | 我们 |
| `trade_no` | 码商订单号 | 码商 |
| `type` | "alipay" / "wxpay" / "qqpay" | 用户选 |
| `name` | 商品名 | 我们(套餐名) |
| `money` | 金额(元,小数) | 我们,按 CNY 小数 |
| `notify_url` | webhook 回调地址 | 我们:`/api/billing/webhook/epay/{channel}` |
| `return_url` | 用户支付完跳回 | 我们:`/dashboard/billing/success` |
| `sign` | MD5 签名 | 双向,都按相同算法 |
| `sign_type` | 固定 "MD5" | — |
| `trade_status` | "TRADE_SUCCESS" = 成功 | 码商 |

**签名算法**(双向一致):
```
sorted_params = {k:v for k,v in params.items() if v not in (None, "") and k not in ("sign", "sign_type")}
raw = "&".join(f"{k}={v}" for k,v in sorted(sorted_params.items()))
sign = md5((raw + merchant_key).encode()).hexdigest()
```
