### 1.1 详细打分依据

**候选 A — Nilou Network(推荐)**
- 信任 5/5: "网络"是技术中性词,中文圈接受度高,无负面联想。
- 国际 5/5: `Nilou` 在英文圈虽不常见,但音节简单(Nee-loo),`Network` 是国际通用词,组合易记。
- 反审查 5/5: 通篇无敏感关键词,微信群聊 / 朋友圈分享不会触发自动审查。
- SEO 4/5: `Nilou` 与原神角色"妮露"重名是 SEO 双刃剑(流量加成 vs 法律风险,见 §5);加 `Network` 后差异化明显。
- GFW 5/5: 不命中任何已知词表。
- **决定性优势**: 域名 `nilou.cc` 已注册并部署,沉没成本利好。

**候选 B — Nilou Cloud**
- 失分点: "云"在中国市场是巨头泛滥词(阿里云、腾讯云、华为云、火山云、青云、七牛云…),用户第一反应是"又一个云厂商",品类混淆;英文圈 `Cloud` 同样过度饱和。
- 反审查 4/5: 中性,但"云"+ "VPN 类服务"在某些 ISP 风控规则里被联动标记。

**候选 C — Nilou Access**
- 失分点: 中文"加速"严重红海(迅游、UU、奇游、雷神…),且这些品牌是国内合规的"游戏加速器",我们走出去会被误归类后又因不合规被打;英文 `Access` 词义过宽。

**候选 D — Nilou Airport**
- 失分点: "机场"是圈内人黑话,圈外人看不懂,圈内人一眼识别 = GFW 关键词扫描器一眼识别。**严禁作为公开品牌名**(可作圈内昵称,但不进官方文案)。

**候选 E — Nilou VPN**
- 失分点: `VPN` 三个字母在 2026 年的中国互联网生态里是"自爆按钮":
  - Apple App Store 中国区直接下架
  - 微信 / QQ 自动撤回
  - 微博 / 小红书自动屏蔽
  - 国内域名注册商秒拒
  - 工信部直接关注
- 即使我们走海外市场,`VPN` 也降低专业感(技术圈普遍认为 `VPN` 是 90 年代企业 IPSec 词,现代隐私服务用 `private network` / `relay` / `tunnel`)。

## 2. 推荐方案 — Nilou Network / 妮露网络

### 2.1 决策理由

1. **零敏感词命中**: 中英文双语在 GFW / Apple / 微信 / 微博 / 国内 ISP 风控的所有公开词表里均不命中。
2. **域名沉没成本**: `nilou.cc` 已注册并完成生产部署,改名 = 抛弃所有已建立的反代 / DNS / 客户引导链路。
3. **国际化平衡**: 英文圈用户能拼能念能记;中文圈用户觉得"像个正经技术品牌"。
4. **品类清晰**: `Network` 暗示"网络基础设施"而非"翻墙工具",符合我们想要的工程师叙事(见 §4)。
5. **法律风险可控**: `Nilou` 单独使用是波斯语"荷花"(نیلو)的拉丁转写,作为通用名词不构成商标侵权;只要不绑定原神视觉素材即可(见 §5)。

### 2.2 弃用方案的退路

如未来出现以下情况,则启动 Plan B(改名):
- 米哈游主动发律师函并主张品牌混淆(概率<5%,因纯名词无视觉绑定)
- `nilou.cc` 域名被 GFW 整体阻断且 CF Tunnel 也无效
- 出现重大法律风险(用户告我们 / 监管点名)

Plan B 候选(预留,本文档不展开):
- `Lotus Relay` / 莲花中继
- `Helix Net` / 螺旋网络
- `Beacon Tunnel` / 灯塔隧道

## 3. Slogan 候选

### 3.1 中文圈(优先级排序)

1. **`抗封 · 透明 · 不跑路`** ⭐ 主推
   - 三短句节奏,直击中国用户三大痛点(被封 / 黑箱 / 跑路);"不跑路"是杀手锏,直接对标行业痛点。
2. `工程师做的机场,代码全开源`
   - 工程师叙事 + 信任锚(开源)。适合在 V2EX / 即刻 / 推特中文圈使用。
3. `源码可查,审计可做,跑路不能`
   - 押韵感、对仗感强;适合做 banner / 海报标题。

### 3.2 英文圈(优先级排序)

1. **`Open-source. Audit-able. Won't disappear.`** ⭐ 主推
   - 三短句对应中文版,语义对齐;`Won't disappear` 隐喻"不像 Mullvad 之外那些跑路服务",在英文隐私圈引发共鸣。
2. `Built by an engineer who hates getting blocked.`
   - 第一人称叙事,符合 indie hacker / Hacker News 文化;适合 GitHub README / 推特 bio。
3. `Censorship-resistant. Source-available. Run by a human.`
   - 对标 Mullvad / IVPN 的隐私品牌叙事;`Run by a human` 在 LLM-everywhere 时代是差异化卖点。

### 3.3 slogan 使用守则

- **不许换着用**: 一段时期内只用一个主 slogan,品牌一致性 > 创意多样性。
- **不进入营销话术**: slogan 出现在 banner / README / 域名 footer 即可,**不出现在客服话术 / 销售文案**(避免变成空话)。
- **每年审视一次**: 在每年 Q4 复盘是否仍准确(产品定位变了 → slogan 跟变)。

## 4. 品牌语调(Voice & Tone)指南

### 4.1 中文圈

**语调**: 工程师叙事 + 透明可查 + 克制不夸张
**词库白名单**:
- ✅ 开源 / 可审计 / 透明 / 工程师 / 自建 / Reality / 抗封 / 不跑路 / 源码 / GitHub
- ✅ "我"(第一人称,显得是一个真人在做)
- ✅ 数字事实("已经稳定运行 X 天"、"X 个节点")

**词库黑名单**:
- ❌ "无限速"、"100% 不被封"、"军用级加密"、"全球最快"(夸大效果)
- ❌ "翻墙"、"科学上网"、"梯子"、"机场"(在公开品牌物料里;圈内黑话不上台面)
- ❌ "原神"、"提瓦特"、"须弥"、"米哈游"、"HoYoverse"(法律红线,见 §5)
- ❌ "我们团队"(目前是单人项目,装大显得不真诚)

### 4.2 英文圈

**语调**: censorship resistance + open-source + dev-friendly,贴近 Mullvad / Tailscale / Cloudflare 的工程师博客风格
**词库白名单**:
- ✅ `censorship-resistant`、`source-available`、`audit-able`、`Reality protocol`、`self-hosted`、`indie-run`
- ✅ 第一人称单数 `I`(强调 indie / solo)
- ✅ 技术术语(SNI、DPI、TLS 1.3、X25519)— 受众是技术人,术语 = 信任

**词库黑名单**:
- ❌ `military-grade encryption`(在密码学圈是 meme,等于自爆专业度)
- ❌ `unlimited speed`、`100% guaranteed`、`unbreakable`(过度承诺)
- ❌ `China VPN`、`GFW bypass`(SEO 关键词诱惑大,但拉低品牌调性,且把自己钉在政治叙事上,反而吸引麻烦)
- ❌ `Genshin`、`miHoYo`、`HoYoverse`(法律红线)

### 4.3 通用禁忌(中英文都适用)

1. **不夸大效果**: 任何"100%"、"绝对"、"永远"、"军用级"、"全球最"都不出现。
2. **不承诺不被封**: 永远只说"抗封"(resistant),不说"不被封"(unblockable)。Reality 协议本身的审计文档也只敢说 resistant。
3. **不暗示与米哈游关联**: 文案中不主动提原神,不引用、不引述、不二创、不暗示。

## 5. 法律红线声明(关键章节)

### 5.1 核心红线

> **严禁使用米哈游(miHoYo Co., Ltd. / HoYoverse Inc.)的任何著作权 / 商标素材。**

具体禁止:
1. ❌ **原神官方角色立绘**(妮露 / Nilou 在《原神》3.1 版本登场,完整 3D 模型 / 立绘 / Q 版形象 / 表情包均为米哈游著作权)
2. ❌ **角色具体形象元素**(妮露的服饰图案、舞蹈姿态、武器形象、技能特效)
3. ❌ **米哈游 / HoYoverse / 原神 / Genshin Impact / Teyvat / Sumeru 的商标组合**
4. ❌ **任何二次创作衍生**(同人图、AI 生图模仿原神画风、cosplay 照片)
5. ❌ **官方配音 / BGM / 角色台词**

### 5.2 风险评估

| 行为 | 风险等级 | 说明 |
|---|---|---|
| 使用 `Nilou` 单词作为品牌名(无视觉) | 🟡 中等可控 | "Nilou" 是波斯语 نیلو(荷花)的拉丁转写,作为通用名词在 60+ 国家有自然分布;米哈游对单字符/单词无独占权(商标只对"原神 / Genshin Impact"等组合) |
| 使用波斯风格 / 荷花元素 logo(原创) | 🟢 低 | 荷花是公共领域元素;原神妮露的服饰是阿拉伯-波斯-印度混合风,但服饰风格本身不可商标化 |
| 使用原神妮露立绘 / 二创图 | 🔴 高 | DMCA / 商标侵权直接命中;米哈游法务在 2024-2025 已对多起同人商业化诉讼成功 |
| 文案中暗示"原神同款"、"提瓦特玩家专属" | 🔴 高 | 商标淡化(trademark dilution)+ 不正当竞争 |
| 域名前缀使用 `genshin-`、`hoyo-`、`teyvat-` | 🔴 极高 | UDRP 流程下米哈游可强制转移域名 |

### 5.3 我们采取的策略

1. **品牌名 `Nilou` 单独使用,不绑定原神视觉**: 在所有官网 / GitHub / 推特 / 文档中,不出现任何原神素材。
2. **logo 100% 原创**: 由本人或委托设计师从零设计,设计稿 / 源文件存档备查;**不使用 AI 生图(避免训练数据污染)**。
3. **关于品牌名出处的官方话术**(如被问到):
   > "Nilou 取自波斯语 نیلو,意为'荷花',象征在浑水中保持清洁。与任何游戏角色或商业作品无关联。"
4. **域名防御注册**:
   - 主: `nilou.cc`(已有)
   - 防御: `nilou.network`(待注,优先级 P0)
   - 防御候选: `nilou.io`、`nilou.dev`、`niloutunnel.com`(预算允许时补)
5. **若收到米哈游函件**:
   - 立即停止争议元素(若有)
   - 律师函交由专业 IP 律师(不自行回复)
   - 启动 Plan B(改名)

### 5.4 DMCA / UDRP / 商标侵权风险点列举

| 风险类型 | 触发条件 | 缓解措施 |
|---|---|---|
| **DMCA takedown**(GitHub / CF / VPS) | 仓库 / 网页含原神官方素材 | 全栈不引用任何米哈游素材 |
| **UDRP**(域名仲裁) | 域名内含 `genshin` / `hoyo` 或恶意关联 | 只用 `nilou` 通用词;不在 whois 暗示游戏关联 |
| **商标侵权**(中国 / 美国) | 在分类 9(软件)/ 38(电信)注册"Nilou Network"被驳 | 注册前做商标检索;若与米哈游已有商标冲突,改用 Plan B |
| **不正当竞争** | 在中文文案中蹭原神热度 | 文案禁用任何原神关键词(见 §4.3) |

## 7. 决策记录(ADR-style)

### 7.1 上下文

项目 aegis-panel 已完成生产部署(域名 `nilou.cc`),进入对外推广前需固化品牌身份。需要决策:正式品牌全名(中英)、slogan、法律红线、视觉资产路径。

### 7.2 决策

采用 **`Nilou Network` / 妮露网络** 为正式品牌名;主 slogan 为"抗封 · 透明 · 不跑路" / `Open-source. Audit-able. Won't disappear.`;严守 §5 法律红线;视觉资产按 §6 清单逐项产出。

### 7.3 备选方案

详见 §1 候选对比表。Plan B(若 Plan A 失败)候选见 §2.2。

### 7.4 后果

**正向**:
- 域名沉没成本归零(`nilou.cc` 继续用)
- 品牌一致性建立,后续所有文案 / 视觉 / 法务有统一基准
- 法律风险面收敛到"不绑定米哈游视觉"这一可执行红线

**负向 / 风险**:
- 与米哈游"妮露"角色重名带来的潜在商标审查复杂度(尤其中国市场)
- 若米哈游未来扩展商标到"网络服务"分类,可能被动改名
- `Nilou` 在英文圈知名度低,需额外品牌建设投入
